Conocimiento en Testing

Evaluación

  1. Una deficiencia en el código de un programa es:

    Un caso de prueba

    Un fallo

    Una funcionalidad

    Error del caso de prueba

  2. Un Retest es:

    Repetir la prueba, cuando se han corregido los errores

    Pruebas de regresión de la funcionalidad

    Pruebas de seguridad de la aplicación

    Verificación de caso de prueba de la funcionalidad

  3. Pruebas de Regresión:

    Comprobación de la idoneidad para el uso esperado

    Conjunto de casos de prueba para un componente

    Condición o capacidad necesaria para un usuario con el objeto de solucionar un problema

    Pruebas de una aplicación previamente probada que ha sufrido modificaciones

  4. Una historia de usuario __________________________________________________________________

    Es un tipo de diagrama de estructura estática que describe la estructura de un sistema mostrando las clases del sistema

    Son descripciones cortas y simples de una funcionalidad

    Conjunto de casos de prueba para un componente o sistema

    Pruebas de software

  5. Su objetivo es detectar errores en datos, lógica y algoritmos:

    Test Unitario

    Test Funcional

    Test de sistema

    Test de Aceptación
